Hello Word Bootstrap

O que é o Bootstrap?

O Bootstrap é uma biblioteca orientada componentes, ou seja parte dos layouts já vem criadas e nos cabe alterá-las para o que desejamos. Agilizando muito a parte de desenvolvimento front-end. Criando também um padrão de grid facilitando a exibição de conteúdo baseado na resolução.

Como se programa com Bootstrap

Precisamos conhecer os componentes, consultar documentação, copiar o componente e excluir o que não vamos utilizar ou criar o nosso a partir da refereência das clases.

Diferentes formas de instalação

No Bootstrap temos formas diferentes de fazer o setup. O CDN é um servidor externo que detém os arquivos estáticos. Os arquivos internos, instalamos o Bootstrap a partir de arquivos no nosso servidor, não é muito interessante. Com o npm que é o gerenciador de pacotes do Node.js, que pode instalar o Bootstrap como depêndencia.

Configurações inportantes

Há dois elementos necessários para o correto funcionamento do Bootstrap que precisamos nos atentar no HTML, o DOCTYPE e o viewport (que controla a responsividade) sem esses elementos ele pode funcionar mas com instabilidades.

O que são breakpoints

São pontos específicos de resolução criados através de media query para deixar o site responsivo. Eles seguem um padrão que é encontrado na maioria sos dispositivos, ou seja, temos um breackpoint para tablets, outro para celulares, não existe para todos, mas para os mais utilizados, lembrando que é utilizado um padrã mobile first, ou seja, estilização de menos resolução para maior.

Os breackpoints do Bootstrap são utilizados por classes como sm, md, lg, cada um representa uma media query e estão normalmente adequamos outros elementos do nosso site a estes breakpoints, assim é possível trabalhar com conjunto de bootstrap deixando o site responsivo.

Containers

Containers são entidades muto importantes do Bootstrap, estruturam o layout e como o nome diz são containers, depositam outros elementos, temos tipos diferentes de containers, vamos ver como funcionam cada um deles, o uso é o mesmo , separar o site em aréas.

Container Fluid

O COntainer Fluid utiliza uma área maior para abrigar os elementos, sempre representa 100% da área útil, eleminando as margens laterais. A class é Container-fluid.

Container por breackpoint

Podemos também moldar um container pelo breackpoint, utilizamos a classe container-x, onde o x pode ser sm, md, lg e outras classes de breackpoint também, com esta versão, queremos um container que respeite uma largura máxima, que se baseia em uma resolução

Grid

O Grid é uma estrutura para criar layouts com bootstrap, temos um grid colunar, dividido em 12 colunas, A estrutura Grid é sempre: container-row colunas, onde temos o container pata abrigar a seção. Row para determinar uma linha (largura de 100%) e colunas para subdividir containers em até 12 partes.

Tamanho das colunas

Podemos ajustar o tamanho das colunas, para ocupar três espaços: col-3, lembrando que podemos separar em no máximo 12 divisões e o espaço ocupado será proporcional, ou seja, em col-11 e col-1 teremos uma coluna ocupando quase 100% da largura da row.
